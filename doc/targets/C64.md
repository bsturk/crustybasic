# crustyBASIC on Commodore 64

Use `@OPTION TARGET c64` for the default C64 profile, or pick an
exact system with `@OPTION SYSTEM`.

Portable runtime calls are documented in [`../API.md`](../API.md).
This page covers C64 systems, modes, formats, and hardware notes. For
command-line target selection and dialects, see
[`../USAGE.md`](../USAGE.md).

## Systems

| System | Default output | Notes |
| --- | --- | --- |
| `c64.orig` | `.prg` | Default. Original NMOS C64 profile. |
| `c64.c64c` | `.prg` | C64C profile. Same feature set here. |
| `c64.ultimate` | `.prg` | C64 Ultimate profile with turbo, UCI, and REU support. |

Loaded `.prg` builds use the resident BASIC V2 ROM for `REAL` by
default. Cartridge builds use crustyBASIC's built-in `REAL` support.

## Basics

| Item | Value |
| --- | --- |
| CPU | 6510, 6502 family |
| Text | 40x25 |
| Code start | `$0810` for `.prg`, `$8000` for cart |
| Program RAM top | `$D000` |
| Text screen | `$0400` |
| Color RAM | `$D800` |
| String encoding | PETSCII |
| Main chips | VIC-II and SID |

`.prg` files load with the normal Commodore `LOAD`, then start with
`SYS`.

## VIC-II Graphics

The `VIC` namespace exposes common VIC-II registers. Use runtime
graphics and sprite calls first; use `VIC.*` when you need direct
hardware control.

Common names:

| Name | Purpose |
| --- | --- |
| `VIC.BORDER` | Border color. |
| `VIC.BACKGROUND`, `VIC.BACKGROUND1`..`3` | Background colors. |
| `VIC.RASTER` | Raster line. |
| `VIC.CTRL1`, `VIC.CTRL2` | Screen control and scroll bits. |
| `VIC.MEMORY` | Screen, bitmap, and character memory selection. |
| `VIC.SPRITE_ENABLE` | Enable sprite bits. |
| `VIC.SPRITE0_X`, `VIC.SPRITE0_Y` | Sprite 0 position. Sprite 1..7 match. |
| `VIC.SPRITE_X_MSB` | High X bits for all sprites. |
| `VIC.SPRITE_PRIORITY` | Sprite priority bits. |
| `VIC.SPRITE_MULTICOLOR` | Sprite multicolor enable bits. |
| `VIC.SPRITE_SPRITE_COLLISION` | Sprite collision latch, cleared on read. |
| `VIC.SPRITE_BG_COLLISION` | Sprite/background collision latch, cleared on read. |

```basic
VIC.BORDER = 0
VIC.BACKGROUND = 6
VIC.SPRITE_ENABLE = 1
VIC.SPRITE0_X = 100
VIC.SPRITE0_Y = 100
```

C64 graphics support text/cell mode, hi-res bitmap, and multicolor
bitmap. The default graphics mode is `BITMAP_HIRES`.

| Mode | C64 mode | Size | Colors |
| --- | --- | --- | --- |
| `CELL` | VIC-II 40 column text | 40x25 | text |
| `BITMAP_HIRES` | VIC-II hi res bitmap | 320x200 | 2 |
| `BITMAP_LORES` | not supported | | |
| `BITMAP_MULTICOLOR` | VIC-II multicolor bitmap | 160x200 | 4 |
| `CELL_MULTICOLOR` | not supported | | |

`@OPTION VICII_BANK` selects a fixed 16 KB video bank by number:

| Value | Video bank | Bitmap | Screen attributes |
| --- | --- | --- | --- |
| `0` | `$0000-$3FFF` | `$2000` | `$0400` |
| `1` | `$4000-$7FFF` | `$6000` | `$4400` |
| `2` | `$8000-$BFFF` | `$A000` | `$8400` |
| `3` | `$C000-$FFFF` | `$E000` | `$C400` |

Bank 1 is the default.

## Images

Native image display supports:

| Format | Accepted files |
| --- | --- |
| `IMAGE_FMT_C64_MULTI` | Koala Painter, 10003 bytes, load address `$6000`, including `$FE` RLE variants |
| `IMAGE_FMT_C64_HIRES` | Art Studio, 9002-9009 bytes, load address `$2000` |

KERNAL-backed runtime image loading is available. A full Koala image
plus program code can exceed the default bitmap bank's code ceiling;
select `@OPTION VICII_BANK 3` for large programs.

## Charset and Tiles

The C64 target supports 8 byte writable text glyphs in VIC-II character
memory. Use `CHARSET_COPY_DEFAULT addr` to copy the ROM font into RAM,
then `CHARSET_INSTALL addr` to select that RAM page. `CHARSET_INSTALL`
only selects the page; it does not copy font bytes.

The default writable charset base used by examples is `$3000` in VIC
bank 0. Cell backed `TILE_DEFINE` installs a RAM charset once, copies
the ROM font first, then writes defined tile glyphs into that charset.
Passing a base to `TILE_BEGIN cell_w, cell_h, base` selects another
2K-aligned charset page.

## Sprites

The C64 has 8 hardware sprites. Portable sprite mode is selected at
compile time:

| `MULTICOLOR_SPRITE` | `SPRITE_WIDTH` x `SPRITE_HEIGHT` | Artwork row | Visible colors | Data |
| --- | --- | --- | --- | --- |
| `FALSE` (default) | 24x21 footprint | 24 one bit cells | 1 | 63 bytes |
| `TRUE` | 24x21 footprint | 12 two bit cells displayed double wide | 3 | 63 bytes |

`SPRITE_WIDTH` remains 24 because sprite coordinates and bounds use the
displayed footprint.

`SPRITE_HAS_INVERT` is `1`. `SPRITE_INVERT` inverts the 63 bytes loaded
into a sprite slot. `SPRITE_EXPAND` can display a sprite at 48x42.
Both modes keep per sprite X and Y expansion, priority, and collision
latches.

`SPRITE_DATA` copies the same 63 target native bytes in either mode. It
does not convert a standard shape into a multicolor shape.

Each multicolor pixel uses one bit pair:

| Bits | Color |
| --- | --- |
| `00` | transparent |
| `01` | shared `SPRITE_COLOR2`, `VIC.SPRITE_MULTICOLOR0`, `$D025` |
| `10` | `SPRITE_COLOR`, `VIC.SPRITE0_COLOR` through `SPRITE7_COLOR`, `$D027` through `$D02E` |
| `11` | shared `SPRITE_COLOR3`, `VIC.SPRITE_MULTICOLOR1`, `$D026` |

`MULTICOLOR_SPRITE TRUE` selects multicolor for every sprite managed by
the portable API. A C64 program can mix modes with
`VICII_SPRITE_MULTICOLOR id, flag` or direct `VIC.SPRITE_MULTICOLOR`
bits. `VICII_SPRITE_MULTICOLOR_COLORS c1, c2` sets the two shared colors.
See [`multicolor_sprite_compare.cbs`](../../examples/c64/crustybasic/multicolor_sprite_compare.cbs)
for the same expanded multicolor shape through the portable and direct APIs.

## SID Sound

Each SID has three voices. Sound uses native SID frequency values. Note
conversion uses MIDI-style note numbers.

Target-specific helpers are available through the `SID.*` namespace:

| Call | Purpose |
| --- | --- |
| `SID.FREQ VOICE, FREQ` | Set native 16-bit frequency. |
| `SID.ADSR VOICE, A, D, S, R` | Set envelope nibbles. |
| `SID.CTRL VOICE, CTRL` | Set waveform and gate bits. |
| `SID.PULSE_WIDTH VOICE, PW` | Set 12-bit pulse width. |
| `SID.GATE_ON VOICE`, `SID.GATE_OFF VOICE` | Toggle the gate bit. |
| `SID.FILTER CUTOFF, RES, VOICES, MODE` | Configure the filter. |
| `SID.MASTER_VOLUME VOLUME` | Set master volume. |
| `SID.OFF VOICE`, `SID.ALL_OFF` | Silence one voice or all voices. |

Useful constants include `TRIANGLE`, `SAW`, `PULSE`,
`NOISE`, `GATE`, and `VOLUME_MAX`.

`SID_CHIPS` accepts 1, 2, 4, or 8, providing 3, 6, 12, or 24 voices.
The first N default bases are `$D400`, `$D420`, `$D480`, `$D4A0`,
`$D500`, `$D520`, `$D580`, and `$D5A0`. Use `SID.SET_BASE` to override
a base for VICE or other hardware:

```basic
@OPTION SYSTEM c64.orig
@OPTION SID_CHIPS 4
SID.SET_BASE 2, $DE00
```

## Input

| Capability | Value |
| --- | --- |
| Keyboard | Yes |
| Individual held keys | Yes |
| Joystick ports | 2 |
| Buttons | 1 per port |
| Stick type | Digital |
| Paddle axes | 4 |
| Keypad ports | 0 |
| Keypad `INPUT` | No |
| Mouse buttons | 0 |

Paddle reads use SID `POTX`/`POTY`. Trigger reads follow the native
active-low convention: `0` means pressed, `1` means released.

`KEY_HELD(code)` scans the keyboard matrix without consuming typed input.
It supports `KEY_SPACE`, `KEY_0` through `KEY_9`, and `KEY_A` through
`KEY_Z`.

## Timing

The runtime tick source is the KERNAL jiffy clock. It runs at about
60 Hz on PAL and NTSC because the KERNAL calibrates the CIA timer by
video standard. The 24-hour wrap is compensated for one crossing.

## C64 Ultimate

Use an exact C64 Ultimate system when you want the Ultimate specific
hardware APIs:

```basic
@OPTION SYSTEM c64.ultimate
```

The C64 Ultimate profile adds:

| API | Purpose |
| --- | --- |
| `SPEED_DETECT`, `SPEED_TURBO`, `SPEED_NORMAL`, `SPEED_SET` | Software controlled turbo. Index 15 is 48 MHz on the C64 Ultimate profile. |
| `EXTMEM_AVAILABLE()`, `EXTMEM_STASH`, `EXTMEM_FETCH`, `EXTMEM_OP` | REU style extended memory DMA. The address is `BANK` plus a 16-bit offset, so bank 255 reaches the 16 MB range. |
| `UCI_AVAILABLE()` | Ultimate Command Interface availability. |
| `FILES_*` | Fast file access through the UCI DOS target. |
| `NET_*` | TCP and UDP access through the UCI network target. |

## Text

Strings encode to PETSCII. Common escapes:

| Escape | Byte |
| --- | --- |
| `{RETURN}`, `{ENTER}` | `$0D` |
| `{TAB}` | Four spaces |
| `{CLEAR}`, `{CLR}` | `$93` |
| `{HOME}` | `$13` |
| `{LOWER}`, `{TEXT_LOWER}` | `$0E` |
| `{UPPER}`, `{TEXT_UPPER}`, `{GRAPHICS}` | `$8E` |
| `{LEFT}`, `{RIGHT}`, `{UP}`, `{DOWN}` | Cursor controls |
| `{RVS_ON}`, `{RVS_OFF}` | Reverse video controls |

## Files

File channels default to disk device 8 and use the logical channel as
the secondary address. C64-specific opens can choose another device or
secondary address.

When building with `@OPTION OUTPUT_TYPE disk` or
`--set output-type=disk`, the D64 wrapper also looks for a companion
directory beside the source named `<program>.disk_files`. Regular files
inside it are copied into the generated disk image as `SEQ` files using
their leaf filenames.

The lower KERNAL channel helpers are also callable by target-specific
programs: `OPEN_CH`, `CLOSE_CH`, `GET_CH`, `PUT_CH`, `PRINT_CH_STR`,
`ST`, and `CMD_CH`.

## Raster

Frame waits poll the VIC raster line. Raster interrupt hooks are not
implemented for C64 yet. Use direct `VIC.RASTER`, `VIC.IRQ_STATUS`, and
`VIC.IRQ_MASK` access for custom raster interrupt work.

## Cartridges

Select cartridge output at build time with `--set output-type=cart`:

```text
crustybasic build <input.cbs> --set output-type=cart -o program.crt
```

Use source-level `OUTPUT_TYPE` only when the program should always build
as a cart:

```basic
@OPTION TARGET c64
@OPTION OUTPUT_TYPE cart
```

Default carts use standard 8K/16K `.crt` output. For larger programs,
select the Super Games mapper:

```basic
@OPTION TARGET c64
@OPTION OUTPUT_TYPE cart
@OPTION MAPPER supergames
```

`supergames` provides four switched 8K banks plus a fixed 8K area.
See [`../USAGE.md#banked-builds`](../USAGE.md#banked-builds) for
`@BANK`.

For EasyFlash output, select `cart-easy-flash`:

```text
crustybasic build <input.cbs> --set output-type=cart-easy-flash -o program.crt
```

```basic
@OPTION TARGET c64
@OPTION OUTPUT_TYPE cart-easy-flash
```

`cart-easy-flash` uses EasyFlash 16K banks. Main code is placed in
physical bank 0. Use `@BANK PRG 0` through `@BANK PRG 62` for switched
banks; those are emitted as physical banks 1 through 63. Switched bank
code cannot call `MAIN`.

## Dialects

`cbm_basic_v2` and `cbm_basic_v7_0` use Commodore suffix rules: bare
numeric names, `!`, and `#` are `REAL`, `%` is 16-bit integer, and `$`
is `STRING`. Use `cbm_basic_v3_5` for Plus/4 BASIC 3.5 listings.

In Commodore BASIC dialects, `RND(X)` returns a `REAL` fraction from
`0` up to, but not including, `1`. Use `RAND(N)` for portable bounded
integers.
