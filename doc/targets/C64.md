# crustyBASIC on Commodore 64

Use `@OPTION TARGET c64` for the default C64 profile, or pick an
exact system with `@OPTION SYSTEM`.

For portable calls such as `CLS`, `POSITION`, `BEEP`, `SOUND`, `JOY`,
`JOY_BUTTON`, `PADDLE`, `PLOT`, `LINE`, and `SPRITE_*`, see
[`../API.md`](../API.md). For command-line target selection and
dialects, see [`../USAGE.md`](../USAGE.md).

## Systems

| System | Output | Notes |
| --- | --- | --- |
| `c64.orig` | `.prg` | Default. Original NMOS C64 profile. |
| `c64.c64c` | `.prg` | C64C profile. Same feature set here. |
| `c64.orig.cart` | `.crt` | C64 cartridge image. |
| `c64.c64c.cart` | `.crt` | C64C cartridge image. |

Loaded `.prg` builds use the resident BASIC V2 ROM for `REAL` by
default. Cartridge builds use crustyBASIC's built-in `REAL` support.

## Basics

| Item | Value |
| --- | --- |
| CPU | 6510, 6502 family |
| Text | 40x25 |
| Code start | `$0810` for `.prg`, `$8000` for cart |
| Program RAM top | `$D000` |
| Text screen | `$0400` |
| Color RAM | `$D800` |
| Main chips | VIC-II and SID |

`.prg` files load with the normal Commodore `LOAD`, then start with
`SYS`.

## VIC-II Graphics

The `VIC` namespace exposes common VIC-II registers. Use the portable
graphics and sprite APIs first; use `VIC.*` when you need direct
hardware control.

Common names:

| Name | Purpose |
| --- | --- |
| `VIC.BORDER` | Border color. |
| `VIC.BACKGROUND`, `VIC.BACKGROUND1`..`3` | Background colors. |
| `VIC.RASTER` | Raster line. |
| `VIC.CTRL1`, `VIC.CTRL2` | Screen control and scroll bits. |
| `VIC.MEMORY` | Screen, bitmap, and character memory selection. |
| `VIC.SPRITE_ENABLE` | Enable sprite bits. |
| `VIC.SPRITE0_X`, `VIC.SPRITE0_Y` | Sprite 0 position. Sprite 1..7 match. |
| `VIC.SPRITE_X_MSB` | High X bits for all sprites. |
| `VIC.SPRITE_PRIORITY` | Sprite priority bits. |
| `VIC.SPRITE_MULTICOLOR` | Sprite multicolor enable bits. |
| `VIC.SPRITE_SPRITE_COLLISION` | Sprite collision latch, cleared on read. |
| `VIC.SPRITE_BG_COLLISION` | Sprite/background collision latch, cleared on read. |

```basic
VIC.BORDER = 0
VIC.BACKGROUND = 6
VIC.SPRITE_ENABLE = 1
VIC.SPRITE0_X = 100
VIC.SPRITE0_Y = 100
```

The portable `DISPLAY`, `PLOT`, `LINE`, and `GFX_COLOR` calls support
text/cell mode, hi-res bitmap, and multicolor bitmap.

## Sprites

The C64 has 8 hardware sprites. Each sprite is 24x21 pixels and uses
63 bytes of bitmap data. The portable `SPRITE_*` API maps to VIC-II
sprites.

| Feature | Value |
| --- | --- |
| Sprite count | 8 |
| Size | 24x21 |
| Data bytes | 63 |
| Stretch | Per-sprite X and Y |
| Collision | Sprite/sprite and sprite/background latches |

Multicolor sprites are available through `VIC.SPRITE_MULTICOLOR`,
`VIC.SPRITE_MULTICOLOR0`, and `VIC.SPRITE_MULTICOLOR1`.

## SID Sound

The SID has three voices. `SOUND VOICE, FREQ, WAVEFORM, VOLUME` starts
a tone using native SID frequency values. `SOUND_FREQ(NOTE)` converts a
MIDI-style note number to that value.

Target-specific helpers are available through the `SID.*` namespace:

| Call | Purpose |
| --- | --- |
| `SID.FREQ VOICE, FREQ` | Set native 16-bit frequency. |
| `SID.ADSR VOICE, A, D, S, R` | Set envelope nibbles. |
| `SID.CTRL VOICE, CTRL` | Set waveform and gate bits. |
| `SID.PULSE_WIDTH VOICE, PW` | Set 12-bit pulse width. |
| `SID.GATE_ON VOICE`, `SID.GATE_OFF VOICE` | Toggle the gate bit. |
| `SID.FILTER CUTOFF, RES, VOICES, MODE` | Configure the filter. |
| `SID.MASTER_VOLUME VOLUME` | Set master volume. |
| `SID.OFF VOICE`, `SID.ALL_OFF` | Silence one voice or all voices. |

Useful constants include `SID_TRIANGLE`, `SID_SAW`, `SID_PULSE`,
`SID_NOISE`, `SID_GATE`, and `SID_VOLUME_MAX`.

## Input

| Capability | Value |
| --- | --- |
| Keyboard | Yes |
| Joystick ports | 2 |
| Buttons | 1 |
| Paddle axes | 4 |
| Mouse buttons | 0 |

`PADDLE(AXIS)` reads SID `POTX`/`POTY`. `PTRIG(AXIS)` follows the
native active-low convention: `0` means pressed, `1` means released.

## Text

Strings encode to PETSCII. Common escapes:

| Escape | Byte |
| --- | --- |
| `{RETURN}`, `{ENTER}` | `$0D` |
| `{TAB}` | Four spaces |
| `{CLEAR}`, `{CLR}` | `$93` |
| `{HOME}` | `$13` |
| `{LOWER}`, `{TEXT_LOWER}` | `$0E` |
| `{UPPER}`, `{TEXT_UPPER}`, `{GRAPHICS}` | `$8E` |
| `{LEFT}`, `{RIGHT}`, `{UP}`, `{DOWN}` | Cursor controls |
| `{RVS_ON}`, `{RVS_OFF}` | Reverse video controls |

## Files

The portable `FILE_OPEN` defaults to disk device 8 and uses the logical
channel as the secondary address. For example, channel 1 maps like
`OPEN 1,8,1,"name,S,R"` for read mode.

Use `FILE_OPEN_NATIVE` when a C64-specific program needs another
device or secondary address:

```basic
@OPTION TARGET c64

FILE_OPEN_NATIVE 1, "DATA,S,W", 9, 1
FILE_WRITE_STR 1, "HELLO"
FILE_CLOSE 1
```

The lower KERNAL channel helpers are also callable by target-specific
programs: `OPEN_CH`, `CLOSE_CH`, `GET_CH`, `PUT_CH`, `PRINT_CH_STR`,
`ST`, and `CMD_CH`. These are not portable API names.

## Raster

`WAIT_FRAME` works by polling the VIC raster line. The portable
`RASTER_*` interrupt API is not implemented for C64 yet. Use direct
`VIC.RASTER`, `VIC.IRQ_STATUS`, and `VIC.IRQ_MASK` access for custom
raster interrupt work.

## Cartridges

Use an exact cartridge system name:

```basic
@OPTION SYSTEM c64.orig.cart
```

Default carts use standard 8K/16K `.crt` output. For larger programs,
select the Super Games mapper:

```basic
@OPTION SYSTEM c64.orig.cart
@OPTION MAPPER supergames
```

`supergames` provides four switched 8K banks plus a fixed 8K area.
See [`../USAGE.md#banked-builds`](../USAGE.md#banked-builds) for
`@BANK`.

## Dialects

`cbm_basic_v2` and `cbm_basic_v7_0` use Commodore suffix rules: bare
numeric names, `!`, and `#` are `REAL`, `%` is 16-bit integer, and `$`
is `STRING`. Use `cbm_basic_v3_5` for Plus/4 BASIC 3.5 listings.

In Commodore BASIC dialects, `RND(X)` returns a `REAL` fraction from
`0` up to, but not including, `1`. Use `RAND(N)` for portable bounded
integers.
