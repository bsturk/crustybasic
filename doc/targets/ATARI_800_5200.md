# crustyBASIC on Atari 8-bit and 5200

The `atari800` and `atari5200` targets share ANTIC, GTIA, POKEY,
Player/Missile Graphics, and a 6502-family CPU. The Atari 800 target
runs as a computer program with the Atari OS. The Atari 5200 target
runs as a cartridge with no computer OS.

For portable calls such as `CLS`, `POSITION`, `WAIT_FRAME`, `SOUND`,
`JOY`, `JOY_BUTTON`, `PLOT`, `LINE`, and `SPRITE_*`, see
[`../API.md`](../API.md).

## Systems

| System | Output | Notes |
| --- | --- | --- |
| `atari800.xl` | `.xex`, optional `.atr` | Default Atari 800 system. XL/XE OS profile. |
| `atari800.orig` | `.xex`, optional `.atr` | Original 400/800 OS profile. |
| `atari800.xl.cart` | `.car` | XL/XE cartridge. |
| `atari800.orig.cart` | `.car` | 400/800 cartridge. |
| `atari5200.orig` | `.a52` | Atari 5200 cartridge. No OS or `REAL`. |

Use `@OPTION TARGET atari800` for the default computer target,
`@OPTION TARGET atari5200` for the console target, or
`@OPTION SYSTEM atari800.xl.cart` for an exact system.

## Basics

| Item | Atari 800 | Atari 5200 |
| --- | --- | --- |
| Text | 40x24 | 40x24 |
| Code start | `$3000` (`$A000` cart) | `$4000` |
| Output | `.xex`, `.car` | `.a52` |
| Keyboard | Yes | No |
| `REAL` | Atari OS math pack | Not supported |
| Graphics | OS `S:` plus direct graphics | Direct graphics |
| Sound | POKEY | POKEY |

Atari text uses ATASCII. Common escapes include `{RETURN}` (`$9B`),
`{CLEAR}` (`125`), cursor directions, and `{TAB}`.

## Graphics

Both targets support the portable graphics API:

| Call | Purpose |
| --- | --- |
| `DISPLAY mode` | Select a graphics mode. |
| `GFX_COLOR fg, bg` | Set drawing colors. |
| `PLOT x, y`, `UNPLOT x, y` | Draw or clear pixels. |
| `LINE x0, y0, x1, y1` | Draw a line. |

Atari 800 also accepts familiar Atari BASIC spellings:

| Spelling | Purpose |
| --- | --- |
| `GRAPHICS mode` | Open an Atari OS graphics mode. |
| `COLOR c` | Set the OS drawing color. |
| `DRAWTO x, y` | Draw from the previous graphics cursor. |
| `LOCATE(x, y)` | Read a graphics value. |

## Player/Missile Graphics

The portable `SPRITE_*` API uses Player/Missile Graphics: four
8x16 player sprites, one color each, with horizontal stretch and
collision flags.

Useful Atari-specific PMG calls:

| Call | Purpose |
| --- | --- |
| `PMGRAPHICS MODE` | Enable or disable PMG. |
| `PMHPOS OBJ, X` | Set horizontal position. |
| `PMPOKE OBJ, Y, BITS` | Write one sprite row. |
| `PMCOLOR OBJ, COLOR` | Set player or missile color. |
| `PMSIZE OBJ, SIZE` | Normal, double, or quad width. |
| `PMHITCLR` | Clear collision latches. |
| `PMHIT_PLAYFIELD OBJ` | Read playfield collision bits. |
| `PMHIT_PLAYER OBJ` | Read player collision bits. |

Objects are `PMG_PLAYER0`..`PMG_PLAYER3` and
`PMG_MISSILE0`..`PMG_MISSILE3`.

## Raster Effects

Use `RASTER_*` for display-list interrupts. They work on both Atari
800 and Atari 5200.

| Call | Purpose |
| --- | --- |
| `RASTER_CLEAR` | Clear all marks. |
| `RASTER_MARK_ROW ROW` | Mark one visible display-list row. |
| `RASTER_MARK_ROWS START, COUNT, STEP` | Mark several rows. |
| `RASTER_MARK_OFFSET OFFSET` | Mark a raw display-list byte offset. |
| `RASTER_INSTALL FRAME_ADDR, LINE_ADDR` | Install frame and line hooks. |
| `RASTER_OFF` | Disable raster hooks. |
| `RASTER_WSYNC` | Wait for the next scanline. |

The installed hooks should be short `@ASYNC` procedures. `RASTER_INDEX`
is reset before the frame hook and increments after each line hook.

Runnable examples:

- [`../../examples/atari800/crustybasic/raster_bars.cbs`](../../examples/atari800/crustybasic/raster_bars.cbs)
- [`../../examples/atari5200/crustybasic/raster_bars.cbs`](../../examples/atari5200/crustybasic/raster_bars.cbs)

## Sound

`SOUND VOICE, PITCH, DISTORTION, VOLUME` drives POKEY. Voices are
`0..3`; volume is `0..15`.

Atari-specific helpers:

| Call | Purpose |
| --- | --- |
| `SET_AUDCTL FLAGS` | Set POKEY audio control. |
| `SOUND_FREQ NOTE` | Get a note divisor. |
| `SOUND16 PAIR, FREQ, DISTORTION, VOLUME` | Use voices 1+2 or 3+4 as a 16-bit pair. |

## Input

| Capability | Atari 800 | Atari 5200 |
| --- | ---: | ---: |
| Keyboard | Yes | No |
| Joystick ports | 4 | 2 |
| Buttons | 1 | 1 |
| Paddle axes | 8 | 4 |
| Analog joystick | No | Yes |

Portable `JOY`, `JOY_BUTTON`, `KEY`, `INKEY`, `ANY_INPUT`, and
`WAIT_ANY_INPUT` work when the hardware exists.

Raw accessors:

| Call | Purpose |
| --- | --- |
| `STICK(N)` | Atari 800 raw stick direction nibble. |
| `STRIG(N)` | Raw trigger: `0` pressed, `1` released. |
| `PADDLE(AXIS)` | Raw paddle or analog controller value. |
| `PTRIG(AXIS)` | Raw paddle trigger, active-low. |

The 5200 has no keyboard. Guard typed-text workflows with
`HAS_KEYBOARD`.

## CIO And Atari 800 OS Calls

These are Atari 800 only.

The portable `FILE_OPEN` maps to CIO defaults. Use `FILE_OPEN_NATIVE`
when an Atari-specific program needs explicit CIO AUX bytes:

```basic
@OPTION TARGET atari800

FILE_OPEN_NATIVE 1, "D:DATA.TXT", A800_CIO_MODE_WRITE, 0
FILE_WRITE_STR 1, "HELLO"
FILE_CLOSE 1
```

For fully Atari-specific code, the CIO helpers below are callable
directly. These are not portable API names. Atari 5200 has no computer
OS file provider.

| Call | Purpose |
| --- | --- |
| `A800_OPEN_CH CH, MODE, AUX2, FILESPEC$` | Open an IOCB channel. |
| `A800_CLOSE_CH CH` | Close a channel. |
| `A800_GET_CH CH`, `A800_PUT_CH CH, BYTE` | Byte I/O. |
| `A800_PRINT_CH_STR CH, S$` | Write a string to a channel. |
| `A800_PUT_DATA CH, COUNT` | Write bytes from `DATA`. |
| `A800_STATUS_CH CH` | Last CIO status. |
| `A800_CIO_XIO CMD, CH, AUX1, AUX2, FILESPEC$` | Raw XIO-style call. |
| `A800_INPUT_REFILL CH` | Refill the input line buffer from a channel. |
| `A800_INPUT_FIELD_NUM`, `A800_INPUT_FIELD_STR` | Parse comma-separated fields. |

Common status constants include `A800_CIO_OK`, `A800_CIO_EOF`, `A800_CIO_BREAK`,
`A800_CIO_NOT_READY`, `A800_CIO_MISSING_DEVICE`, `A800_CIO_IOCB_NOT_OPEN`, and
`A800_CIO_FILE_NOT_FOUND`.

Other Atari 800 calls:

| Call | Purpose |
| --- | --- |
| `A800_DOS` | Exit to DOS. |
| `A800_BYE` | Exit through the OS. |
| `A800_STOP` | Stop the program and return through the Atari exit path. |
| `CLR` | Clear user globals. |
| `A800_LPRINT_STR`, `A800_LPRINT_EOL` | Printer output on `P:`. |
| `A800_LOG10(X)` | Common log through the Atari math pack. |

## Hardware Names

Direct chip namespaces are available for hardware work:

| Namespace | Purpose |
| --- | --- |
| `GTIA` | Colors, triggers, collisions, PMG registers. |
| `POKEY` | Audio, paddles, random byte, keyboard code. |
| `ANTIC` | Display list, character base, scroll, scanline waits. |
| `PIA` | Atari 800 joystick ports and control registers. |

On the 5200, the same chip names use the 5200 memory map.

## Cartridges

Atari 800 cartridge systems default to standard 8K/16K carts. Use
`@OPTION MAPPER xegs32` for a 32K XEGS-style banked cart.

```basic
@OPTION SYSTEM atari800.xl.cart
@OPTION MAPPER xegs32
```

See [`../USAGE.md#banked-builds`](../USAGE.md#banked-builds) for
`@BANK` syntax.

Atari 5200 cartridges use the console BIOS splash screen by default.
Set the raw final two year glyph bytes shown by the BIOS with
`ATARI_5200_BIOS_YEAR`. The BIOS supplies the leading `19`; digit
glyphs are ASCII plus `$20`, so `$5959` renders as `1999`.

```basic
@OPTION TARGET atari5200
@OPTION ATARI_5200_BIOS_YEAR $1234
```

The high byte is written at `$BFFC`; the low byte is written at
`$BFFD`. Avoid a low byte of `$FF` unless you intentionally want the
BIOS diagnostic-cart path.

## Dialects

`atari_basic` and `basic_xl` use Atari suffix rules: bare numeric
names, `!`, and `#` are `REAL`, `%` is 16-bit integer, and `$` is
`STRING`.

In `atari_basic`, `RND(X)` returns a `REAL` fraction from `0` up to,
but not including, `1`. The argument is accepted for compatibility.
Use `RAND(N)` for portable bounded integers.
