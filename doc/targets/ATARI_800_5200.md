# crustyBASIC on Atari 8-bit and 5200

The `atari800` and `atari5200` targets share ANTIC, GTIA, POKEY,
Player/Missile Graphics, and a 6502-family CPU. The Atari 800 target
runs as a computer program with the Atari OS. The Atari 5200 target
runs as a cartridge with no computer OS.

Portable runtime calls are documented in [`../API.md`](../API.md).
This page covers Atari systems, modes, formats, and hardware notes.

## Systems

| System | Output | Notes |
| --- | --- | --- |
| `atari800.orig` | `.xex`, optional `.atr` | Original 400/800 OS profile. |
| `atari800.xl` | `.xex`, optional `.atr`; `.car` with cart output | Default Atari 800 system. XL/XE OS profile. |
| `atari800.xe` | `.xex`, optional `.atr`; `.car` with cart output | XE OS profile. |
| `atari800.xegs` | `.car` | Atari XEGS cartridge system. XE-family OS profile. |
| `atari5200.orig` | `.a52` | Atari 5200 cartridge. No OS or `REAL`. |

Use `@OPTION TARGET atari800` for the default computer target,
`@OPTION TARGET atari5200` for the console target, or
`@OPTION SYSTEM atari800.xegs` for the XEGS cartridge system.

## Basics

| Item | Atari 800 | Atari 5200 |
| --- | --- | --- |
| Text | 40x24 | 40x24 |
| Code start | `$3000` (`$A000` cart) | `$4000` |
| Output | `.xex`, `.car` | `.a52` |
| Keyboard | Yes | No |
| `REAL` | Atari OS math pack | Not supported |
| String encoding | ATASCII | ATASCII |
| Graphics | OS `S:` plus direct graphics | Direct graphics |
| Sound | POKEY | POKEY |

Atari text uses ATASCII. Common escapes include `{RETURN}` (`$9B`),
`{CLEAR}` (`125`), cursor directions, and `{TAB}`.

## Memory Regions

Run `crustybasic target-info <system>` to see the memory regions available
for that system, including their ranges, uses, loading limits, and whether
they are enabled automatically or must be enabled by the program. Enabling
a region lets the compiler place supported code or data in that RAM.

Enable a region in source:

```basic
@OPTION MEMORY_REGION region_name
```

Or in a program config file:

```toml
memory-regions = ["region_name"]
```

## Disk Files

When building Atari 800 programs with `@OPTION OUTPUT_TYPE disk` or
`--set output-type=disk`, the ATR wrapper looks for a companion
directory beside the source named `<program>.disk_files`. Regular files
inside it are copied into the generated DOS 2 disk image using their
leaf filenames.

## Graphics

Both targets support the portable graphics modes below. Both targets
also expose Atari `GRAPHICS_*` constants for native ANTIC/GTIA display
modes. See [`../API.md`](../API.md) for drawing calls.

Atari 800 uses the Atari OS `S:` screen handler through CIO. Atari 5200
has no computer OS or CIO screen handler, so it builds display lists and
writes text or bitmap memory directly.

Atari 800 mode mapping:

The default graphics mode is `BITMAP_MULTICOLOR`.

| Mode | Atari 800 mode | Size | Colors |
| --- | --- | --- | --- |
| `CELL` | `GRAPHICS 0` | 40x24 | text |
| `BITMAP_HIRES` | `GRAPHICS 8 + 16` | 320x192 | 2 |
| `BITMAP_LORES` | `GRAPHICS 5 + 16` | 80x48 | 4 |
| `BITMAP_MULTICOLOR` | `GRAPHICS 11 + 16` | 80x192 | 16 |
| `CELL_MULTICOLOR` | not supported | | |

On Atari 800, `GFX_PLOT_COLOR_AVAILABLE` is `TRUE` only for Graphics 11
and its full-screen form. Other modes use shared color registers, so
three-argument `PLOT` falls back to `PLOT(x, y)`.

`DISPLAY_MIXED(mode, 4)` selects the normal Atari BASIC graphics form
with the four line bottom text window. `DISPLAY_MIXED(mode, 0)` selects
the full screen form, the same as `DISPLAY(mode)` for shared bitmap
modes. Other text window row counts are not supported by the OS mode
provider.

`DISPLAY 1` is the shared `BITMAP_HIRES` request, not Atari
`GRAPHICS 1`. Runtime mode selection should use shared modes or the
`GRAPHICS_*` constants. For example, `DISPLAY GRAPHICS_2`
selects Atari OS `GRAPHICS 2` 40x12 text. `GRAPHICS` accepts either a
raw Atari mode number, such as `GRAPHICS 17`, or one of the
`GRAPHICS_*` constants, such as `GRAPHICS GRAPHICS_1`.

Atari 5200 portable mode mapping:

The default graphics mode is `BITMAP_MULTICOLOR`.

| Mode | Atari 5200 mode | Size | Colors |
| --- | --- | --- | --- |
| `CELL` | direct text display list | 40x24 | text |
| `BITMAP_HIRES` | ANTIC mode F bitmap | 320x192 | 2 |
| `BITMAP_LORES` | ANTIC mode A bitmap | 80x48 | 4 |
| `BITMAP_MULTICOLOR` | GTIA 16 color bitmap | 80x96 | 16 |
| `CELL_MULTICOLOR` | not supported | | |

On Atari 5200, `GFX_PLOT_COLOR_AVAILABLE` is `TRUE` for the portable
multicolor mode and native Graphics 9 through 11. It is `FALSE` for
the portable hires and lores modes.

Atari 5200 also supports `DISPLAY GRAPHICS_0` through
`DISPLAY GRAPHICS_15` and the full-screen `GRAPHICS_*_FULL`
constants listed below. Text modes `GRAPHICS_0`, `GRAPHICS_1`,
`GRAPHICS_2`, `GRAPHICS_12`, and `GRAPHICS_13` update `PRINT`,
`POSITION`, `CLS`, and charmap addressing for the active text width and
height. Bitmap modes support `PLOT`, `POINT`, `UNPLOT`, and line
drawing through direct bitmap memory.

On Atari 5200, normal bitmap `GRAPHICS_*` modes use the Atari BASIC
mode dimensions but do not create a bottom OS text window. Use the
`GRAPHICS_*_FULL` constants for full-height bitmap display lists.

In CrustyBASIC source for Atari 800 and Atari 5200, `GRAPHICS` is
accepted directly:

| Spelling | Purpose |
| --- | --- |
| `GRAPHICS mode` | Open an Atari graphics mode from a raw number or `GRAPHICS_*` constant. |

For Atari 5200, `GRAPHICS mode` builds the equivalent display list
directly. For Atari 800, it opens the OS screen channel through CIO.

In Atari 800 CrustyBASIC source, and in Atari dialects, these Atari
BASIC spellings are also accepted directly:

| Spelling | Purpose |
| --- | --- |
| `COLOR c` | Set the OS drawing color. |
| `SETCOLOR reg, hue, lum` | Set a color shadow from Atari hue and luminance nibbles. |
| `SETCOLOR reg, color` | Set a color shadow from a packed target color such as `LIGHT_GREEN`. |
| `DRAWTO x, y` | Draw from the previous graphics cursor. |
| `FCOLOR c` | Set the OS fill color. |
| `FILLTO x, y` | Use the OS fill-to command. |
| `LOCATE(x, y)` | Read a graphics value. |

`SETCOLOR` takes one of the five Atari OS color shadow selectors listed
under register aliases below. There are no `SETCOLOR` selectors beyond
slot `4`. Player/missile color shadows are `PCOLR0` through `PCOLR3`;
the direct GTIA registers are exposed as `GTIA.COLPM0` through
`GTIA.COLPM3`.

The Atari 800 keyboard shadow `CH` is exposed as `CH_ADDR` so programs
can clear it with `POKEB CH_ADDR, CH_NONE` without colliding with local
character variables.

The two-argument `SETCOLOR` form accepts the same packed target color
bytes used by shared color constants such as `BLACK`, `ORANGE`, and
`LIGHT_GREEN`; see the named colors in [`../API.md`](../API.md#graphics).
Use the three-argument form when you need to mix hue and luminance
separately.

In Atari 800 `GRAPHICS 0` and Atari 5200 `GRAPHICS_0` or `CELL` text,
foreground and background hues are not fully independent. Text and
graphics color requests map this as a best effort: PF1 supplies
foreground luminance, PF2 supplies foreground hue plus background
luminance, and COLOR4/BAK supplies the border/background color. Use
`SETCOLOR` on Atari 800 or direct `GTIA` writes on either target for
exact target-specific display-list effects.

In Atari 800 and Atari 5200 CrustyBASIC source, Atari graphics
constants are available for `DISPLAY` and `GRAPHICS`.
`DISPLAY GRAPHICS_8` and `GRAPHICS GRAPHICS_8` select the same Atari

| `GRAPHICS_2` | `GRAPHICS 2` | 20x12 | 5 text |
| `GRAPHICS_3` | `GRAPHICS 3` | 40x20 | 4 |
| `GRAPHICS_4` | `GRAPHICS 4` | 80x40 | 2 |
| `GRAPHICS_5` | `GRAPHICS 5` | 80x40 | 4 |
| `GRAPHICS_6` | `GRAPHICS 6` | 160x80 | 2 |
| `GRAPHICS_7` | `GRAPHICS 7` | 160x80 | 4 |
| `GRAPHICS_8` | `GRAPHICS 8` | 320x160 | 2 |
| `GRAPHICS_9` | `GRAPHICS 9` | 80x160 | 16 |
| `GRAPHICS_10` | `GRAPHICS 10` | 80x160 | 9 |
| `GRAPHICS_11` | `GRAPHICS 11` | 80x160 | 16 |
| `GRAPHICS_12` | `GRAPHICS 12` | 40x24 | 5 text |
| `GRAPHICS_13` | `GRAPHICS 13` | 40x12 | 5 text |
| `GRAPHICS_14` | `GRAPHICS 14` | 160x160 | 2 |
| `GRAPHICS_15` | `GRAPHICS 15` | 160x160 | 4 |
| `GRAPHICS_3_FULL` | `GRAPHICS 3+16` | 40x24 | 4 |

## Charset and Tiles

Both Atari targets support 8 byte writable text glyphs through ANTIC
`CHBASE`. Use `CHARSET_COPY_DEFAULT addr` to copy the default font into
RAM, then `CHARSET_INSTALL addr` to select that RAM page.
`CHARSET_INSTALL` only selects the page; it does not copy font bytes.

The Atari 800 default writable charset base is `$7000`. The Atari 5200
default writable charset base is `$1C00`; its default font is copied
from the console ROM at `$F800`.

Cell backed `TILE_DEFINE` installs a RAM charset once, copies the
default font first, then writes defined tile glyphs into that charset.
Passing a base to `TILE_BEGIN cell_w, cell_h, base` selects another
1K-aligned charset page.

### Register aliases

Atari OS shadow register selectors and addresses use their OS names.
Use the chip namespaces for direct hardware writes, such as
`ANTIC.DLISTL`, `ANTIC.DLISTH`, `GTIA.COLPF0`, or `GTIA.COLBK`.

| Alias | Value | OS shadow | Hardware register |
| --- | --- | --- | --- |
| `SDLSTL` | `$0230` | `SDLSTL` / `SDLSTH` | `ANTIC.DLISTL` / `ANTIC.DLISTH`; write with `DPOKE` |
| `PCOLR0` | `$02C0` | `PCOLR0` | `GTIA.COLPM0` |
| `PCOLR1` | `$02C1` | `PCOLR1` | `GTIA.COLPM1` |
| `PCOLR2` | `$02C2` | `PCOLR2` | `GTIA.COLPM2` |
| `PCOLR3` | `$02C3` | `PCOLR3` | `GTIA.COLPM3` |
| `PF0` | `0` | `COLOR0` (`$02C4`) | `GTIA.COLPF0` |
| `PF1` | `1` | `COLOR1` (`$02C5`) | `GTIA.COLPF1` |
| `PF2` | `2` | `COLOR2` (`$02C6`) | `GTIA.COLPF2` |
| `PF3` | `3` | `COLOR3` (`$02C7`) | `GTIA.COLPF3` |
| `BORDER` | `4` | `COLOR4` (`$02C8`) | `GTIA.COLBK` |
| `ATACHR` | `$02FB` | `ATACHR` | S: character/color byte |
| `CH_ADDR` | `$02FC` | `CH` | keyboard code shadow |
| `CH_NONE` | `$FF` | no key | byte used to clear `CH_ADDR` |

## Images

Both targets support the same native image formats:

| Format | Accepted files |
| --- | --- |
| `IMAGE_FMT_ATARI_MIC` | Micro Painter or raw Graphics 15 dumps, 7680/7684/7685 bytes |
| `IMAGE_FMT_ATARI_GR8` | Raw Graphics 8 dumps, 7680/7682 bytes |

Atari 800 can also stream `.img` descriptors through OS files. Atari
5200 cartridge builds can display embedded descriptors but do not have
file loading.

## Player/Missile Graphics

Sprites use all eight Player/Missile Graphics objects. They share one
ID space:

| Sprite ID | Target identifier | Data |
| --- | --- | --- |
| `0`..`3` | `PMG_PLAYER0`..`PMG_PLAYER3` | Custom 8x16 shape |
| `4`..`7` | `PMG_MISSILE0`..`PMG_MISSILE3` | Fixed narrow four-row shape |

`SPRITE_MAX_COUNT` is `8`; the PMG identifiers above are aliases for
those IDs, not additional objects. All eight IDs support
`SPRITE_MOVE`, `SPRITE_COLOR`, `SPRITE_SHOW`, and `SPRITE_HIDE`.
`SPRITE_DATA` is limited to the four player IDs, as reported by
`SPRITE_HAS_DATA`. `SPRITE_WIDTH`, `SPRITE_HEIGHT`, and
`SPRITE_DATA_BYTES` describe those player shapes.

Player N and missile N share the same color register. Changing either
slot's color also changes its paired object, as reported by
`SPRITE_COLOR_SHARED`. `SPRITE_NAME` returns the target identifiers
shown in the table.

Useful Atari-specific PMG calls:

| Call | Purpose |
| --- | --- |
| `PMGRAPHICS MODE` | Enable or disable PMG. |
| `PMHPOS OBJ, X` | Set horizontal position. |
| `PMPOKE OBJ, Y, BITS` | Write one sprite row. |
| `PMCOLOR OBJ, COLOR` | Set player or missile color. |
| `PMSIZE OBJ, SIZE` | Normal, double, or quad width. |
| `PMHITCLR` | Clear collision latches. |
| `PMHIT_PLAYFIELD OBJ` | Read playfield collision bits. |
| `PMHIT_PLAYER OBJ` | Read player collision bits. |

The PMG calls and `SPRITE_*` calls access the same objects. For example,
`SPRITE_MOVE PMG_MISSILE0, X, Y` moves sprite ID `4`.

## Raster Effects

Display-list interrupts work on both Atari 800 and Atari 5200. The
installed hooks should be short `@ASYNC` procedures. The raster index
is reset before the frame hook and increments after each line hook.
See [`../API.md`](../API.md#raster-interrupts) for the call names.

## Timing

| Target | Tick source | Notes |
| --- | --- | --- |
| Atari 800 | OS `RTCLOK` | Free running. True rate is about 59.92 Hz NTSC or 49.86 Hz PAL. |
| Atari 5200 | BIOS `RCLKHI:RCLKLO` | Free running. One tick per vertical blank, about 59.92 Hz. |

Timer metadata reports 60 Hz on NTSC builds and 50 Hz on PAL Atari 800
builds. Both tick sources are free running.

## Sound

Sound drives POKEY. Voices are `0..3`; volume is `0..15`.

Atari-specific helpers:

| Call | Purpose |
| --- | --- |
| `SET_AUDCTL FLAGS` | Set POKEY audio control. |
| `SOUND16 PAIR, FREQ, DISTORTION, VOLUME` | Use voices 1+2 or 3+4 as a 16-bit pair. |

## Input

| Capability | Atari 800 | Atari 5200 |
| --- | --- | --- |
| Keyboard | Yes | No |
| Individual held keys | Yes | No |
| Joystick ports | 2 on XL/XE, 4 on original 400/800 | 2 |
| Buttons | 1 per port | 1 per port |
| Stick type | Digital | Analog |
| Paddle axes | 4 on XL/XE, 8 on original 400/800 | 4 |
| Keypad ports | 0 | 1 |
| Keypad `INPUT` | No | Unsigned integer |
| Mouse buttons | 0 | 0 |

Raw accessors:

| Call | Purpose |
| --- | --- |
| `STICK(N)` | Atari 800 raw stick direction nibble. |
| `STRIG(N)` | Raw trigger: `0` pressed, `1` released. |
| Paddle axes | Raw paddle or analog controller value. |
| `PTRIG(AXIS)` | Raw paddle trigger, active-low. |
| `KEYPAD_CODE(0)` | Atari 5200 keypad code. |

On Atari 800, `KEY_HELD(code)` reads the current POKEY keyboard state
without consuming typed input. It supports `KEY_SPACE`, `KEY_0` through
`KEY_9`, and `KEY_A` through `KEY_Z`.

The 5200 keypad is exposed as one keypad port. `KEYPAD_CODE(0)` returns
`KEYPAD_0` through `KEYPAD_9`, `KEYPAD_ASTERISK`, `KEYPAD_POUND`,
`KEYPAD_START`, `KEYPAD_PAUSE`, `KEYPAD_RESET`, or `KEYPAD_NONE`.

The 5200 has no keyboard, but `INPUT` accepts unsigned integer variables
from the keypad. Press `#` to finish entry. `*` is available through
`KEYPAD_CODE(0)` and is ignored by numeric `INPUT`. Guard typed-text
workflows with `KEYBOARD_AVAILABLE`.

## CIO And Atari 800 OS Calls

These are Atari 800 only.

Atari 800 CrustyBASIC accepts Atari BASIC-style channel output. This is
useful for the screen channel opened by `GRAPHICS`, and for code that
writes to explicit CIO channels:

```basic
GRAPHICS 17
POSITION 2, 11
PRINT #6; "READY";
PRINT #6
PUT #6, 155
```

| Spelling | Purpose |
| --- | --- |
| `OPEN #ch, mode, aux2, spec$` | Open an Atari CIO channel. |
| `CLOSE #ch` | Close a channel. |
| `GET #ch, var` | Read one byte. |
| `STATUS #ch, var` | Read the last channel status. |
| `PRINT #ch; item; item` | Write items to a channel, then Atari EOL. |
| `PRINT #ch; item;` | Write items without Atari EOL. |
| `? #ch; item` | Short form of `PRINT #ch; item`. |
| `PRINT #ch` | Write Atari EOL. |
| `PUT #ch, byte` | Write one byte. |
| `PUT DATA #ch, count` | Write `count` bytes from the current `DATA` cursor. |

File I/O maps to CIO defaults. Atari-specific native opens can pass
explicit CIO AUX bytes.

Atari 5200 has no computer OS file provider.

## Hardware Names

Direct chip namespaces are available for hardware work:

| Namespace | Purpose |
| --- | --- |
| `GTIA` | Colors, triggers, collisions, PMG registers. |
| `POKEY` | Audio, paddles, random byte, keyboard code. |
| `ANTIC` | Display list, character base, scroll, scanline waits. |
| `PIA` | Atari 800 joystick ports and control registers. |

On the 5200, the same chip names use the 5200 memory map.

## Cartridges

Atari 800 cartridge systems default to standard 8K/16K carts. Use
`@OPTION MAPPER xegs32` for a 32K XEGS-style banked cart.

```basic
@OPTION SYSTEM atari800.xegs
@OPTION MAPPER xegs32
```

See [`../USAGE.md#banked-builds`](../USAGE.md#banked-builds) for
`@BANK` syntax.

Atari 5200 cartridges use the console BIOS splash screen by default.
Set the one-line game name shown by the BIOS with `A5200_GAME_NAME`.
The name is limited to 20 characters and is padded with blanks.

```basic
@OPTION TARGET atari5200
@OPTION A5200_GAME_NAME "MY GAME"
```

Set the raw final two year glyph bytes shown by the BIOS with
`A5200_BIOS_YEAR`. The BIOS supplies the leading `19`; digit
glyphs are ASCII plus `$20`, so `$5959` renders as `1999`.

```basic
@OPTION TARGET atari5200
@OPTION A5200_BIOS_YEAR $1234
```

The high byte is written at `$BFFC`; the low byte is written at
`$BFFD`. Avoid a low byte of `$FF` unless you intentionally want the
BIOS diagnostic-cart path.

## Assembler

Atari 800 and Atari 5200 builds use `vasm6502_oldstyle` through `vasm`.
Release bundles do not include the assembler binary. Place it at
`tools/assemblers/vasm/<platform>/vasm6502_oldstyle` beside the compiler,
using the `.exe` suffix on Windows, or pass its path with `--assembler`.

## Dialects

`atari_basic` and `basic_xl` use Atari suffix rules: bare numeric
names, `!`, and `#` are `REAL`, `%` is 16-bit integer, and `$` is
`STRING`.

In `atari_basic`, `RND(X)` returns a `REAL` fraction from `0` up to,
but not including, `1`. The argument is accepted for compatibility.
Use `RAND(N)` for portable bounded integers.
