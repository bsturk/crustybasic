# crustyBASIC on Commodore 128

Use `@OPTION TARGET c128` for the default Commodore 128 profile. It
builds a BASIC-loaded `.prg` for native C128 mode.

Portable runtime calls are documented in [`../API.md`](../API.md).
This page covers C128 systems, modes, formats, and hardware notes. For
command-line target selection and dialects, see
[`../USAGE.md`](../USAGE.md).

## Systems

| System | Default output | Notes |
| --- | --- | --- |
| `c128.orig` | `.prg` | Default. C128 native mode with BASIC 7.0 and KERNAL ROMs. |

Loaded `.prg` builds use the resident BASIC 7.0 ROM for `REAL` by
default. Cartridge builds use crustyBASIC's built-in `REAL` support.

## Basics

| Item | Value |
| --- | --- |
| CPU | 8502, with Z80 select exposed through the MMU |
| Text | 40x25 VIC-IIe and 80x25 VDC |
| Code start | `$1C10` for `.prg`, `$8000` for cart |
| Program RAM top | `$8400` |
| Text screen | `$0400` |
| Color RAM | `$D800` |
| String encoding | PETSCII |
| Main chips | VIC-IIe, VDC, SID, MMU |

The current target stays in native C128 mode. It does not use `GO64`.

## VIC-IIe Graphics

The `VIC` namespace exposes the VIC-IIe register set:

| Name | Purpose |
| --- | --- |
| `VIC.BORDER` | Border color. |
| `VIC.BACKGROUND` | Background color. |
| `VIC.SPRITE_ENABLE` | Enable sprite bits. |
| `VIC.SPRITE_X_MSB` | High X bits for all sprites. |
| `VIC.SPRITE0_X` through `VIC.SPRITE7_Y` | Sprite positions. |
| `VIC.SPRITE0_COLOR` through `VIC.SPRITE7_COLOR` | Sprite colors. |
| `VIC.SPEED` | 1 MHz / 2 MHz control. |

C128 graphics support VIC-IIe text/cell mode, VIC-IIe bitmap modes, and
VDC 80 column text and 640x200 monochrome bitmap modes. The default
graphics mode is `VICII_BITMAP_HIRES`.

| Mode | C128 mode | Size | Colors |
| --- | --- | --- | --- |
| `VICII_TEXT` | VIC-IIe 40 column text | 40x25 | text |
| `VICII_BITMAP_HIRES` | VIC-IIe hi-res bitmap | 320x200 | 2 |
| `VICII_BITMAP_HIRES_TEXT` | VIC-IIe hi-res bitmap with default text window | 320x160 | 2 |
| `VICII_BITMAP_MULTICOLOR` | VIC-IIe multicolor bitmap | 160x200 | 4 |
| `VICII_BITMAP_MULTICOLOR_TEXT` | VIC-IIe multicolor bitmap with default text window | 160x160 | 4 |
| `VDC_TEXT` | VDC 80 column text | 80x25 | text |
| `VDC_BITMAP_HIRES` | VDC monochrome bitmap | 640x200 | 2 |

The portable display mode constants are also accepted where they map
cleanly to C128 hardware:

| Mode | C128 mode | Size | Colors |
| --- | --- | --- | --- |
| `CELL` | VIC-IIe 40 column text | 40x25 | text |
| `BITMAP_HIRES` | VIC-IIe hi-res bitmap | 320x200 | 2 |
| `BITMAP_LORES` | not supported | | |
| `BITMAP_MULTICOLOR` | VIC-IIe multicolor bitmap | 160x200 | 4 |
| `CELL_MULTICOLOR` | not supported | | |

The default target uses VIC-IIe bank 1 (`$4000-$7FFF`), with screen
attributes at `$5800` and bitmap data at `$6000`. Select `@OPTION
VICII_BANK 2` for bank 2 (`$8000-$BFFF`), with screen attributes at
`$8400` and bitmap data at `$A000`.

`DISPLAY_MIXED(mode, text_rows)` is supported for
`VICII_BITMAP_HIRES`, `VICII_BITMAP_MULTICOLOR`, and the
matching portable bitmap constants. It sets a bottom VIC-IIe text window
and reports only the drawable bitmap area in `GFX_HEIGHT`. `text_rows`
may be `0` through `DISPLAY_MIXED_MAX_ROWS`. The
`VICII_BITMAP_HIRES_TEXT` and
`VICII_BITMAP_MULTICOLOR_TEXT` constants use
`DISPLAY_MIXED_DEFAULT_ROWS`, which is `5`. The BASIC 7.0 dialect maps
`GRAPHIC 2` and `GRAPHIC 4` split modes through this API.

`GFX_PAGE_COUNT`, `GFX_PAGE_SELECT`, `GFX_PAGE_BASE`, and
`GFX_PAGE_ATTR_BASE` are available through the portable graphics API. On
C128 they return one VIC-IIe page in normal graphics modes. In 80 column
mode they expose two VDC text pages, including separate text and
attribute bases. In `VDC_BITMAP_HIRES` they expose one 16000 byte
bitmap page on 16K VDC systems and four monochrome bitmap pages on 64K
VDC systems.

## Charset and Tiles

The C128 target supports 8 byte writable glyphs for VIC-IIe 40 column
text through the portable `CHARSET_*` API. Use `CHARSET_COPY_DEFAULT
addr` to copy the ROM font into RAM, then `CHARSET_INSTALL addr` to
select that RAM page. `CHARSET_INSTALL` only selects the page; it does
not copy font bytes. The default writable charset base used by tile code
is `$3000`.

In cell mode, `TILE_DEFINE` writes each defined glyph to both the
VIC-IIe RAM charset and the VDC character pattern table, so the same
tile definitions work on the 40 and 80 column text paths. Passing a base
to `TILE_BEGIN cell_w, cell_h, base` selects another 2K-aligned VIC-IIe
charset page; VDC tile glyphs use `VDC.CHARSET_DEFAULT_BASE`.

## VDC 80 Column Text

The `VDC` namespace exposes the indirect VDC ports and helpers:

| Name | Purpose |
| --- | --- |
| `VDC.ADDR`, `VDC.STATUS`, `VDC.DATA` | Raw VDC port registers. |
| `VDC.STATUS_RETRACE` | VDC status bit for the retrace window. |
| `VDC.DISPLAY_START_HI`, `VDC.DISPLAY_START_LO` | Internal VDC display start registers. |
| `VDC.UPDATE_ADDR_HI`, `VDC.UPDATE_ADDR_LO` | Internal VDC update address registers. |
| `VDC.ATTRIBUTE_START_HI`, `VDC.ATTRIBUTE_START_LO` | Internal VDC attribute start registers. |
| `VDC.COLOR`, `VDC.ADDRESS_INCREMENT`, `VDC.CHARACTER_BASE` | Internal VDC control registers. |
| `VDC.WORD_COUNT`, `VDC.DATA_REG`, `VDC.BLOCK_START_HI`, `VDC.BLOCK_START_LO` | Internal VDC block and data registers. |
| `VDC.WAIT_VBLANK()` | Wait for the next VDC retrace window. |
| `VDC.WRITE REG, VALUE` | Write an internal VDC register. |
| `VDC.READ(REG)` | Read an internal VDC register. |
| `VDC.POKE ADDR, VALUE` | Write VDC video RAM. |
| `VDC.PEEK(ADDR)` | Read VDC video RAM. |
| `VDC.FILL ADDR, COUNT, VALUE` | Fill VDC video RAM. |
| `VDC.STREAM_WRITE SRC_ADDR, DST_ADDR, COUNT` | Copy CPU memory to VDC RAM with per byte ready polling. |
| `VDC.STREAM_FILL DST_ADDR, COUNT, VALUE` | Fill VDC RAM through the safe streaming path. |
| `VDC.BURST_WRITE SRC_ADDR, DST_ADDR, COUNT` | Experimental no wait stream write for frame timed VDC updates. |
| `VDC.BURST_FILL DST_ADDR, COUNT, VALUE` | Experimental no wait fill for frame timed VDC updates. |
| `VDC.COPY SRC, DST, COUNT` | Copy bytes inside VDC video RAM. |
| `VDC.TEXT_CLEAR PAGE, CH, ATTR` | Clear one 80x25 text page. |
| `VDC.TEXT_POKE PAGE, COL, ROW, CH, ATTR` | Write a character and attribute. |
| `VDC.TEXT_WRITE PAGE, COL, ROW, TEXT$, ATTR` | Write a string and attributes. |
| `VDC.TEXT_COPY SRC_PAGE, DST_PAGE` | Copy one 80x25 text page to another. |
| `VDC.TEXT_PAGE PAGE` | Display VDC text page 0 or 1. |
| `VDC.BITMAP_ON()` / `VDC.BITMAP_OFF()` | Enter VDC bitmap or VDC text mode. |
| `VDC.BITMAP_COLOR FG, BG` | Set VDC bitmap foreground and background colors. |
| `VDC.BITMAP_PAGE PAGE` | Display VDC bitmap page 0, or pages 0 to 3 on 64K VDC. |
| `VDC.BITMAP_PAGE_COUNT()` | Return available VDC bitmap pages. |
| `VDC.BITMAP_PAGE_BASE(PAGE)` | Return the VDC RAM base for a bitmap page. |
| `VDC.RAM_KB()` | Return detected VDC RAM size, `16` or `64`. |
| `VDC.HAS_64K()` | Detect 64K VDC RAM by probing whether `$4000` aliases `$0000`. |
| `VDC.CHARSET_DEFAULT_BASE` | Default VDC character pattern base, `$2000`. |
| `VDC.CHARSET_COPY_DEFAULT BASE` | Copy the default 8 byte glyphs into VDC 16 byte character slots. |
| `VDC.CHARSET_INSTALL BASE` | Select the VDC character pattern base. |
| `VDC.CHARSET_DEFINE CODE, ADDR` | Copy one 8 byte glyph into a VDC character slot. |
| `VDC.CHARSET_RESET()` | Restore the default VDC character pattern base. |

`TEXT_SELECT_80()` selects the 80 column editor,
`TEXT_SELECT_40()` selects the 40 column editor, and
`TEXT_IS_80()` reports the current editor screen. `GRAPHIC 5` in
the `cbm_basic_v7_0` dialect maps to 80 column text.

The two built in VDC pages use `$0000/$0800` and `$1000/$1800` inside
VDC RAM for text and attributes.

`VDC.BURST_*` skips per byte VDC ready polling after setup. Use it only
from a known safe VDC retrace window, usually after `VDC.WAIT_VBLANK()`,
and keep counts small enough.

`DISPLAY(VDC_BITMAP_HIRES)` configures a 640x200 two color bitmap
where `GFX_ROW_BYTES()` returns `80`. `GFX_CLS`, `GFX_COLOR`, `PLOT`, `UNPLOT`,
`POINT`, `HLINE`, `UNHLINE`, and the higher level line, box, and circle
helpers use VDC VRAM through the graphics API. `GFX_BITMAP_BASE()` and
`GFX_PAGE_BASE(page)` return VDC RAM offsets, not CPU addresses.

## Images

Native image display supports C64 compatible VIC-IIe bitmap descriptors:

| Format | Description |
| --- | --- |
| `IMAGE_FMT_C64_MULTI` | Koala Painter, 10003 bytes, load address `$6000`, including `$FE` RLE variants |
| `IMAGE_FMT_C64_HIRES` | Art Studio, 9002-9009 bytes, load address `$2000` |

Runtime image file loading is not available.

## MMU and CPU

The `MMU` namespace exposes the C128 MMU registers at `$D500` through
`$D50B`, including `MMU.CONFIG`, `MMU.MODE_CONFIG`, `MMU.RAM_CONFIG`,
`MMU.PAGE0_LO`, `MMU.PAGE0_HI`, `MMU.PAGE1_LO`, `MMU.PAGE1_HI`, and
read only `MMU.VERSION`.

Low level helpers are available. MMU page arguments are 256 byte page
selectors, not byte addresses. Page `$20` maps address `$2000`; page
`$100` maps banked address `$10000` on hardware that exposes that MMU
address bit.

| Call | Purpose |
| --- | --- |
| `MMU.CONFIG_SET VALUE` | Set the current memory configuration through `$FF00`. |
| `MMU.CONFIG_CURRENT()` | Read the current memory configuration through `$FF00`. |
| `MMU.CPU_SELECT CPU_8502` | Select the 8502 side of the MMU CPU bit. |
| `MMU.CPU_SELECT CPU_Z80` | Select the Z80 side of the MMU CPU bit. |
| `MMU.CPU_CURRENT()` | Read the current MMU CPU bit. |
| `MMU.CPU_IS_8502()` | Report whether the MMU CPU bit selects 8502. |
| `MMU.CPU_IS_Z80()` | Report whether the MMU CPU bit selects Z80. |
| `MMU.CHAR_ROM_MAP` | Map character ROM at `$D000` and switch out I/O. |
| `MMU.CHAR_ROM_UNMAP` | Restore I/O at `$D000`. |
| `MMU.CHAR_ROM_PEEK(OFFSET)` | Temporarily map character ROM, read one byte, then restore the previous config. |
| `MMU.CHAR_ROM_COPY OFFSET, DST, COUNT` | Temporarily map character ROM, copy bytes to RAM, then restore the previous config. |
| `MMU.PAGE0_MAP PAGE`, `MMU.PAGE1_MAP PAGE` | Raw zero page and stack page remap. |
| `MMU.PAGE0_CURRENT()`, `MMU.PAGE1_CURRENT()` | Read the current zero page or stack page selector. |
| `MMU.PAGE0_DEFAULT`, `MMU.PAGE1_DEFAULT` | Restore the zero page or stack page window to page `$00` or `$01`. |
| `MMU.PAGE0_PEEK`, `MMU.PAGE0_POKE`, `MMU.PAGE0_FILL` | Temporary zero page window operations. |
| `MMU.PAGE1_PEEK`, `MMU.PAGE1_POKE`, `MMU.PAGE1_FILL` | Temporary stack page window operations. |
| `MMU.PAGE0_COPY_FROM PAGE, SRC`, `MMU.PAGE1_COPY_FROM PAGE, SRC` | Copy 256 bytes from RAM into the remapped page. |
| `MMU.PAGE0_COPY_TO PAGE, DST`, `MMU.PAGE1_COPY_TO PAGE, DST` | Copy 256 bytes from the remapped page into RAM. |

CrustyBasic currently emits 8502 code for C128. The Z80 select constant
and helper are low level hardware controls, not Z80 code generation or a
CP/M runtime.

## Sprites

The C128 target uses eight VIC-IIe hardware sprites.

| `MULTICOLOR_SPRITE` | `SPRITE_WIDTH` x `SPRITE_HEIGHT` | Artwork row | Visible colors | Data |
| --- | --- | --- | --- | --- |
| `FALSE` (default) | 24x21 footprint | 24 one bit cells | 1 | 63 bytes |
| `TRUE` | 24x21 footprint | 12 two bit cells displayed double wide | 3 | 63 bytes |

`SPRITE_WIDTH` remains 24 because sprite coordinates and bounds use the
displayed footprint.

`SPRITE_DATA_8X8` expands an eight byte shape into a centered 24x21
hardware sprite. `SPRITE_EXPAND` can display a sprite at 48x42. Both
modes keep per sprite X and Y expansion, priority, and collision latches.

`SPRITE_DATA` copies the same 63 target native bytes in either mode. In
multicolor data, `00` is transparent, `01` is shared `SPRITE_COLOR2` at
`$D025`, `10` is per sprite `SPRITE_COLOR` at `$D027+id`, and `11` is
shared `SPRITE_COLOR3` at `$D026`.

`MULTICOLOR_SPRITE TRUE` selects multicolor for every sprite managed by
the portable API. A C128 program can mix modes with
`VICII_SPRITE_MULTICOLOR id, flag` or direct `VIC.SPRITE_MULTICOLOR`
bits.

## SID Sound

The shared SID calls and constants are documented under
[C64 SID Sound](C64.md#sid-sound). The C128 target supports one SID at
`$D400` and does not use `SID_CHIPS`.

Timed sound scheduling is available through the portable
`SOUND_TIMED_*` API. The C128 provider uses the shared chained IRQ
dispatcher, uses 60 Hz `TICKS` units, and returns from timed note calls
immediately.

The portable `EVENT_*` API maps `EVENT_SPRITE_SPRITE`,
`EVENT_SPRITE_BG`, and `EVENT_LIGHTPEN` to VIC-IIe IRQ sources.
Subscribed handlers must be no argument `@ASYNC` PROCs and run from the
same shared IRQ dispatcher as timed sound.

The `cbm_basic_v7_0` dialect includes a synchronous compiled `PLAY`
path for note letters, rests, note lengths, dotted notes, sharps/flats,
`V/O/T/X/U/M` music string commands, `TEMPO`, and shared
envelope/filter state. Notes and rests are scheduled through the C128
timed sound provider, so different SID voices can overlap. `PLAY` still
parses the music string in the foreground and waits when reusing an
active voice or processing `M`.

## Input

| Capability | Value |
| --- | --- |
| Keyboard | Yes |
| Individual held keys | Yes |
| Joystick ports | 2 |
| Buttons | 1 per port |
| Stick type | Digital |
| Paddle axes | 4 |
| Keypad ports | 0 |
| Keypad `INPUT` | No |
| Mouse buttons | 0 |

Paddle reads use SID `POTX`/`POTY`. Trigger reads follow the native
active-low convention: `0` means pressed, `1` means released.

`KEY_HELD(code)` scans the C128 keyboard matrix without consuming typed
input. It supports `KEY_SPACE`, `KEY_0` through `KEY_9`, and `KEY_A`
through `KEY_Z`.

In the `cbm_basic_v7_0` dialect, `JOY(1)` and `JOY(2)` use BASIC 7's
one based joystick numbering and return compass direction values with
128 added when fire is pressed. `POT(1)` through `POT(4)` return raw
paddle values with 256 added when the matching trigger is pressed.
`PEN(0)` and `PEN(1)` read the VIC-IIe light pen position; `PEN(2)`,
`PEN(3)`, and `PEN(4)` read the VDC light pen horizontal position,
vertical position, and trigger state.

## Timing

Delay is available through the runtime throttle. `TICKS` reports a 60 Hz
free-running frame counter advanced by `FRAME_WAIT`. `TIMER_AVAILABLE`
is false, so the elapsed timer API is not available.

`FRAME_WAIT`, `FRAME_INSTALL`, and `FRAME_UNINSTALL` are available
through a synthesized raster wait hook.

## Text

Strings encode to PETSCII. Common escapes:

| Escape | Byte |
| --- | --- |
| `{RETURN}`, `{ENTER}` | `$0D` |
| `{TAB}` | Four spaces |
| `{CLEAR}`, `{CLR}` | `$93` |
| `{HOME}` | `$13` |
| `{LOWER}`, `{TEXT_LOWER}` | `$0E` |
| `{UPPER}`, `{TEXT_UPPER}`, `{GRAPHICS}` | `$8E` |
| `{LEFT}`, `{RIGHT}`, `{UP}`, `{DOWN}` | Cursor controls |
| `{RVS_ON}`, `{RVS_OFF}` | Reverse video controls |

The cell API writes directly to the 40x25 screen at `$0400` and color
RAM at `$D800`. `CELL_CODE` converts ASCII-style characters to screen
codes.

## Files

File channels default to disk device 8 and use the logical channel as
the secondary address. C128-specific opens can choose another device or
secondary address.

The lower KERNAL channel helpers are also callable by target-specific
programs: `OPEN_CH`, `CLOSE_CH`, `GET_CH`, `PUT_CH`, `PRINT_CH_STR`,
`ST`, and `CMD_CH`.

## Cartridges

Select cartridge output at build time with `--set output-type=cart`:

```text
crustybasic build <input.cbs> --set output-type=cart -o program.bin
```

Use source-level `OUTPUT_TYPE` only when the program should always build
as a cart:

```basic
@OPTION TARGET c128
@OPTION OUTPUT_TYPE cart
```

Default carts use native 16K raw `.bin` external function ROM output at
`$8000`. The startup stays in C128 mode and runs through the C128
function ROM path. C64 style cartridges are a separate format and
normally switch the machine into C64 mode.

## Dialects

`cbm_basic_v7_0` targets `c128` by default. It supports Commodore
suffix rules and rewrites the C128 BASIC 7.0 graphics commands
`GRAPHIC`, `SCNCLR`, `COLOR`, `CHAR`, `LOCATE`, `DRAW`, `BOX`,
`CIRCLE`, `PAINT`, `WIDTH`, and `SCALE`; sound commands `VOL`, `SOUND`,
`ENVELOPE`, and `FILTER`; a synchronous compiled `PLAY` subset for
note letters, rests, and note lengths plus `TEMPO`; a compiled
`COLLISION` subset for
enabling or disabling VIC collision and lightpen event sources; sprite
commands `SPRITE`, `MOVSPR`, `SPRCOLOR`, and `SPRSAV`; `BANK` for
`PEEK` and `POKE`; graphics readback functions `RCLR`, `RDOT`, and
`RGR`; input readback functions `JOY`, `POT`, and `PEN`; sprite
readback functions `RSPCOLOR`, `RSPPOS`, `RSPRITE`, and `BUMP`; text
window commands `WINDOW` and `RWINDOW`; and speed
commands `FAST` and `SLOW` to native C128 runtime helpers.

`GRAPHIC 5` switches to 80 column text. `GRAPHIC 2` and `GRAPHIC 4`
map to `DISPLAY_MIXED` and accept the `s` split line argument.

`WIDTH 1` selects single width graphics drawing. `WIDTH 2` selects a
compiled double width path for `DRAW`, `BOX`, and line segments emitted
by advanced circle and arc forms.

`SCALE 1` selects BASIC 7.0 scaled graphics coordinates using 1023 by
1023 in standard bitmap modes and 511 by 511 in multicolor bitmap
modes. `SCALE 1, XMAX, YMAX` selects explicit coordinate ranges. Bare
`SCALE` disables scaling. Scaling affects compiled graphics coordinates
and `MOVSPR`; `RDOT` and `RSPPOS` report the scaled state.

In Commodore BASIC dialects, `RND(X)` returns a `REAL` fraction from
`0` up to, but not including, `1`. Use `RAND(N)` for portable bounded
integers.
